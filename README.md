# Photoshoot Production Scheduler

A full-stack web app for managing commercial photoshoot schedules and daily call sheets.

## Tech Stack

- **Frontend:** React 18 + TypeScript + Tailwind CSS + React Query + Zustand + @dnd-kit
- **Backend:** Node.js + Express + Prisma ORM + PostgreSQL
- **Auth:** JWT (access token 15min + refresh token 7 days, httpOnly cookies)
- **Export:** ExcelJS (.xlsx)
- **Hosting:** Render (Web Service + PostgreSQL + Static Site)

---

## Local Development

### Prerequisites
- Node.js 20+
- PostgreSQL running locally (or use Docker)

### 1. Clone and install

```bash
git clone <your-repo-url>
cd my-project

# Install backend deps
cd backend && npm install

# Install frontend deps
cd ../frontend && npm install
```

### 2. Configure environment

```bash
cd backend
cp .env.example .env
```

Edit `.env`:
```
DATABASE_URL=postgresql://postgres:password@localhost:5432/photoshoot_scheduler
JWT_SECRET=a-long-random-string-here
JWT_REFRESH_SECRET=another-long-random-string-here
NODE_ENV=development
PORT=3000
CLIENT_URL=http://localhost:5173
```

### 3. Set up database

```bash
cd backend
npx prisma migrate dev --name init
npx prisma generate
```

### 4. Start development servers

In two terminals:

```bash
# Terminal 1 — Backend
cd backend
npm run dev

# Terminal 2 — Frontend
cd frontend
npm run dev
```

Frontend runs at http://localhost:5173  
Backend API at http://localhost:3000

### 5. First run

Navigate to http://localhost:5173/register to create the first user (who becomes the Organisation OWNER).

## Render Deployment

### One-click deploy

1. Fork this repository to your GitHub account
2. Go to [render.com](https://render.com) and create a new account or log in
3. Click **New → Blueprint** and connect your GitHub repo
4. Render will detect `render.yaml` and create:
   - `photoshoot-scheduler-api` — Node.js web service
   - `photoshoot-scheduler-web` — Static site (React frontend)
   - `photoshoot-scheduler-db` — PostgreSQL database

### Environment variables to set manually

After deploy, go to the API service dashboard and update:
- `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`, `SMTP_FROM` — for invite and password reset emails
- `CLIENT_URL` — set to your actual frontend URL

`JWT_SECRET` and `JWT_REFRESH_SECRET` are auto-generated by Render.

### First-run instructions

1. Navigate to your frontend URL
2. Click "Create one" to register
3. The first registered user becomes the **Organisation OWNER**
4. Go to Settings → Team to invite team members

## How to Invite Team Members

1. Log in as OWNER or ADMIN
2. Go to **Settings → Team**
3. Click **Invite Member**
4. Enter their email and choose a role:
   - **VIEWER** — read-only access
   - **EDITOR** — can create/edit projects and shots
   - **ADMIN** — full access, can manage users
5. They'll receive an email with a link to set their password

---

## Key Features

- **Multi-organisation:** Each registration creates a new org; all data is isolated
- **Schedule Builder:** Visual grid with sections/categories/locations/shots; click cells to assign shots to shooting days
- **Call Sheet Editor:** Autosaves every 500ms; customisable crew/client/logistics fields with drag-to-reorder
- **Excel Export:** Full schedule workbook with colour-coded header rows, frozen panes, and outline grouping; per-day call sheet export
- **Role-based access:** OWNER > ADMIN > EDITOR > VIEWER
- **iPad-friendly:** All tap targets ≥ 44px
